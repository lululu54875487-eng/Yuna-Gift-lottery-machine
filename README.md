# 小U0贈物抽獎機

小U0贈物抽獎機是一個給社群贈物、名單抽籤、多人撞號抽選使用的小工具。  
把名單貼上後，就可以自動抽出每個贈物號碼的得主，也支援 VIP 指定、每人中獎數限制、全形轉半形與疑似重複檢查。

## 主要功能

### 贈物抽獎

適合多人填選多個贈物號碼的情境。

範例：

```text
璐璐：1、16、23
抽抽：20、15
芊芊：1、15
```

如果只有一個人選某個號碼，系統會直接指定給該人。  
如果多人選到同一個號碼，系統會自動抽出其中一位得主。

### 單純抽籤

如果只是想從一串名單裡抽出一位中獎者，也可以直接貼名字。

範例：

```text
璐璐、抽抽、芊芊
```

也可以用換行分隔：

```text
璐璐
抽抽
芊芊
```

### VIP 指定得主

可以指定某個贈物號碼直接給某位得主，不參加隨機抽選。

範例：

```text
21：璐璐
35：抽抽
```

有填在 VIP 指定欄位的號碼，會直接分配給指定的人。

### 每人中獎數限制

可以設定每個人最多中幾樣：

- 1 樣
- 2 樣
- 3 樣
- 4 樣
- 5 樣
- 6 樣
- 7 樣
- 8 樣
- 9 樣
- 10 樣
- 不限

當某個人已經達到中獎上限時，後續的隨機抽選會自動排除該人。  
如果某個號碼的所有填選者都已經達到上限，該號碼會列在「未抽出號碼」。

### VIP 是否佔用中獎數

可以用核取方塊選擇：

- 勾選：VIP 指定贈物也算進每人中獎數。
- 取消勾選：VIP 指定贈物不佔用一般抽獎的中獎數。

如果想嚴格控制每個人最多拿幾樣，建議勾選。  
如果 VIP 是額外福利，不想影響一般抽獎，可以取消勾選。

### 全形轉半形

如果清單裡有全形數字、全形符號或輸入法造成的格式差異，可以使用「全形轉半形」按鈕整理格式。

例如：

```text
璐璐：１、１６、２３
```

可以轉成較穩定的格式：

```text
璐璐:1,16,23
```

這能減少因格式不同造成的解析錯誤。

### 檢查疑似重複

「檢查疑似重複」按鈕會幫忙找出可能因為以下原因造成的重複項目：

- 全形與半形不同
- 英文大小寫不同
- 多餘空格
- 輸入法符號差異

抽獎前先檢查，可以降低重複登記或漏看的機率。

### 結果複製

抽獎完成後，可以一鍵複製結果，方便貼到社群、留言區或表單紀錄。

結果會包含：

- 每位得主拿到的號碼
- 每位得主共中幾樣
- 各號碼得主對照
- 未抽出號碼
- 當次使用的中獎限制設定

### 跳過動畫

如果贈物很多，不想等動畫跑完，可以按「跳過動畫」快速完成抽選。

### 清空結果與清空全部

- 清空結果：只清除抽獎結果，保留輸入名單。
- 清空清單＋結果：清除所有輸入內容與抽獎結果。

## 支援格式

贈物抽獎可使用：

```text
名字：號碼、號碼、號碼
名字:號碼,號碼,號碼
名字➡️號碼、號碼、號碼
名字->號碼,號碼,號碼
```

單純抽籤可使用：

```text
名字、名字、名字
名字,名字,名字
名字
名字
名字
```

VIP 指定可使用：

```text
號碼：名字
號碼:名字
號碼➡️名字
號碼->名字
```

## 使用方式

1. 開啟 `index.html`。
2. 把贈物清單或抽籤名單貼到主輸入框。
3. 如有 VIP 指定，填到 VIP 指定欄位。
4. 視需要設定每人最多中幾樣。
5. 視需要選擇 VIP 指定是否佔用中獎數。
6. 可先使用「全形轉半形」與「檢查疑似重複」整理名單。
7. 按下「開始抽獎」。
8. 抽完後複製結果貼到社群。

## GitHub 使用建議

這是一個純前端小工具，不需要安裝套件。

建議檔案結構：

```text
Yuna-Gift-lottery-machine/
├── index.html
├── README.md
└── SOCIAL_POST.md
```

上傳到 GitHub 後，可以開啟 GitHub Pages，讓大家直接用網址使用抽獎機。

## 社群貼文

可直接使用 [`SOCIAL_POST.md`](./SOCIAL_POST.md) 裡的版本貼到社群。
